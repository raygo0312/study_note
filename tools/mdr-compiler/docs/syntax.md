# 基本構文

このプロジェクトで作る Markdown 系言語の最初の仕様です。まずは
コンパイラ、シンタックスハイライト、フォーマッタで共有できる構文だけを
定義し、パーサーの実装時にこの仕様を実行例とテストへ落とし込みます。

## 用語の定義

`*` で囲んだ文字列を用語の定義として扱い、`<dfn>`要素へ変換します。

```mdr
これは *重要な文章* です。
```

```html
これは <dfn>重要な用語</dfn> です。
```

斜体は提供しません。したがって、`_文章_` や `*文章*` を斜体として
解釈することはありません。`*文章*` は常に用語の定義です。

太字の範囲内に未対応の入れ子構文を持ち込まず、まずは単純なインライン
テキストを対象にします。

## リンク

リンクはMarkdownと同じ記法です。ラベル内ではMDRのインライン構文を
使用できます。

```mdr
[参照ページ](/reference.html)
[*重要な参照*](#見出し)
```

## エスケープ

MDRの構文記号を文字として書く場合は、直前にバックスラッシュを置きます。

```mdr
\*用語ではない\*
\$数式ではない\$
```

## 見出しと段落

見出しは Markdown と同じく、行頭の `#` の数でレベルを表します。

```mdr
# 大見出し
## 中見出し
### 小見出し
```

見出し以外の通常行は段落として `<p>` に変換されます。空行までの連続した
通常行は同じ段落になり、空行があると次の段落になります。

```mdr
これは一つ目の段落です。
この行も同じ段落です。

これは二つ目の段落です。
```

## コード

インラインコードはバッククォート1組で囲み、複数行のコードはMarkdownと
同じく3個のバッククォートで囲みます。コード内のMDR構文は解釈しません。

```mdr
これは `*そのまま表示する*` です。

```js
const value = 1;
```

```mdr
$A <-> B$

## リスト

行頭の `+` は番号付きリストを表します。番号はソースに書かず、出力時に
`1` から連番で生成します。

```mdr
+ 最初の項目
+ 次の項目
+ 最後の項目
```

```html
<ol>
  <li>最初の項目</li>
  <li>次の項目</li>
  <li>最後の項目</li>
</ol>
```

`+` から始まる連続した行は同じ番号付きリストにまとめます。空行または
別のブロック要素でリストを終了します。

リストは2文字以上のインデントで入れ子にできます。

通常の箇条書きには行頭の `-` を使います。

```mdr
- 選択肢 A
- 選択肢 B
```

`*` を行頭の箇条書き記号としては扱いません。これにより、太字記法との
役割を明確にします。

## タグ

段落内で明示的に改行する場合は、行末へバックスラッシュを書きます。

```mdr
一行目\
二行目
```

これはHTMLの`<br>`として出力されます。

HTMLタグをMDRソースへ直接書かず、汎用ブロックタグで表します。`.名前`は
class、`#名前`はidで、両者の順序は問いません。

```mdr
:::section.def#example 命題
本文です。
:::
```

引数は空白で区切り、空白を含む1引数は`(重要な 命題)`のように囲みます。
引数名と出力属性は、別のMDRファイルで定義してimportできます。

```mdr
@tag section(data-label)
@import "tags.mdr"
```

`input`などのHTML void要素は閉じる`:::`を書かず、一行で完結します。

```mdr
@tag input(type, placeholder)
:::input#searchInput text 検索したい用語を入力
```

`@import`のパスは、記述しているページの場所にかかわらず、プロジェクトの
`src/mdr`ディレクトリを基準にした相対パスです。

ページ固有のクライアントscriptはfrontmatterの`scripts`へ、MDRページからの
相対パスで記述します。生成されたAstroページのローカルscript importとなり、
Astro/Viteによってブラウザ用JavaScriptへ変換・bundleされます。

```mdr
scripts:
  - ../scripts/search.ts
```

インラインタグは`:span[text]`、class付きなら`:span.note[text]`と書きます。
先頭の`:`により、Markdownリンク`[label](destination)`とは区別されます。
内容には別のインラインタグも記述できます。

```mdr
:ruby[Hilbert:rt[ヒルベルト]]
```
