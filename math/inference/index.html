真理関数

命題論理や一階述語論理で一般に定義される真理関数を用意する．用意する真理関数は否定 negation ，連言 conjunction ，選言 disjunction ，含意 implication である．

用意

論理式$\phi,\psi$について，
否定
$$\lnot\paren{\phi}$$
連言
$$\land\paren{\phi,\psi}$$
選言
$$\lor\paren{\phi,\psi}$$
含意
$$\implies\paren{\phi,\psi}$$

形式化する上では論理体系の意味を考えないが，意味がわからないと理解しづらいと思うので，参考までに自然言語としての意味を載せる．直感でわかりやすいように，真理関数は自然言語でよくつかわれる言葉が元になっている．

真理関数の意味

論理式$\phi,\psi$について，
  • 否定$\lnot\paren{\phi}$ : $\phi$ではない
  • 連言$\land\paren{\phi,\psi}$ : $\phi$かつ$\psi$
  • 選言$\lor\paren{\phi,\psi}$ : $\phi$または$\psi$
  • 含意$\implies\paren{\phi,\psi}$ : $\phi$ならば$\psi$

このままでは扱いづらいので，真理関数を演算子として扱いたい．このとき，論理演算子 logical operation ともいう．次のように表す．

定義

論理式$\phi,\psi$としたとき，
$\lnot$演算子
$$\paren{\lnot\phi}\Def\lnot\paren{\phi}$$
$\lor$演算子
$$\paren{\phi\land\psi}\Def\land\paren{\phi,\psi}$$
$\land$演算子
$$\paren{\phi\lor\psi}\Def\lor\paren{\phi,\psi}$$
$\implies$演算子
$$\paren{\phi\implies\psi}\Def\implies\paren{\phi,\psi}$$

二項演算子におけるこの記法を中置記法 infix notation という．一方，前に置く記法を前置記法 prefix notation (ポーランド記法 porlish notation )という．後ろに置く記法を後置記法 postfix notation (逆ポーランド記法 reverse polish notation )という．便利なのでこれらのほかにもう2つ演算子を定義する．

定義

論理式$\phi,\psi$について，
同値
$$\paren{\phi\iff\psi}\Def\paren{\paren{\phi\implies\psi}\land\paren{\psi\implies\phi}}$$
排他的論理和
$$\paren{\phi\veebar\psi}\Def\paren{\phi\land\paren{\lnot\psi}\lor\paren{\paren{\lnot\phi}\land\psi}}$$

排他的論理和 exclusive disjunction はどちらか一方のみであるという意味である．

2つの量化表現を導入する．全称量化子 universal quantifier と全称量化子 universal quantifier である．

定義

全称量化子
$$\forall$$
存在量化子
$$\exists$$

具体的には，論理式$F$に対して「すべての$x$について$F\paren{x}$である」を$\forall x\paren{F\paren{x}}$と表し，全称量化 universal quantification という．また，「$F\paren{x}$となる$x$が存在する」を$\exists x\paren{F\paren{x}}$と表し，存在量化 existential quantification という．

  2. $\exists x\paren{f\paren{x}\gt0}$

演算子の優先順位

論理演算子を用いると，「$P$でないかつ$Q$のときのみ$P$ならば$P$または$Q$である」という曖昧で複雑な命題に対しても$\paren{\paren{\paren{\lnot P}\land Q}\iff P}\implies\paren{P\lor Q}$と命題同士の関係を明確に記述することが出来る．しかし，括弧が冗長で式が煩雑である．そこで，演算子に優先順位 precedence を定める．演算子の優先順位とは括弧がない場合に優先して計算される順番のことである．右結合 right-associative は優先順位が同じ時に，右側の演算子を優先することである．左側の場合は左結合 left-associative である．

定義

論理式$\phi$について，
全体括弧の省略
$$\phi\Def\paren{\phi}$$
$a,b,c$の演算子$\rho_1$が$\rho_2$より優先順位が高いとき，
優先順位
$$a\,\rho_1\,b\,\rho_2\,c\Def\paren{a\,\rho_1\,b}\,\rho_2\,c$$
$a,b,c$の演算子$\rho_1$と$\rho_2$の優先順位が同じで左結合であるとき，
左結合
$$a\,\rho_1\,b\,\rho_2\,c\Def\paren{a\,\rho_1\,b}\,\rho_2\,c$$
$a,b,c$の演算子$\rho_1$と$\rho_2$の優先順位が同じで右結合であるとき，
右結合
$$a\,\rho_1\,b\,\rho_2\,c\Def a\,\rho_1\,(b\,\rho_2\,c)$$
ここでいう論理式は最長一致であることに注意してね
つまり，$\phi\Def\paren{\phi}$をつかって，次のように変形するのはダメだよ
$$a\land\paren{b\lor c}\Iff a\land b\lor c$$

実際に使われる論理演算子の優先順位を定める．

定義

上から順に優先順位が高いとする．また，特に明記しない限り演算子は右結合とする．
  • $\lnot$
  • $\veebar$
  • $\land$
  • $\lor$
  • $\implies$
  • $\iff$

また，各種演算子ごとの大まかな優先順位も定める．推論文字($\Implies,\Iff,\sequent,\consequent$)の優先順位は最下位とする，算術演算とは($+,-,\cup$など)数字や集合など同士の演算子である．

定義

上から順に優先順位が高いとする．
  • 算術演算子
  • 関係演算子
  • 論理演算子

優先順位の例

命題$P,Q,R$について，
    $$\lnot P\land Q\Iff\paren{\lnot P}\land Q$$ $$P\land Q\lor Q\land R\Iff\paren{P\land Q}\lor\paren{Q\land R}$$ $$\lnot\lnot P\Iff\lnot\paren{\lnot P}$$ $$P\lor Q\lor R\Iff\paren{P\lor Q}\lor R$$ $$P\implies Q\implies R\Iff P\implies\paren{Q\implies R}$$ $$\lnot\paren{P\lor Q}\land R\Iff\paren{\lnot\paren{P\lor Q}}\land R$$
練習問題1

次の括弧が省略された命題に括弧をつけろ．ただし，式全体の括弧は省略してよい．

  1. $\lnot P\land Q$
  2. $P\land Q\lor R\implies P\lor\lnot Q\land R$
  3. $P\implies Q\implies P\iff Q$
解答

優先順位が高い演算子から括弧をくくる．同順位の場合は右結合なら右側から，左結合なら左結合がから順に括弧をくくる．

  1. $\paren{\lnot P}\land Q$
  2. $\paren{\paren{P\land Q}\lor R}\implies\paren{P\lor\paren{\paren{\lnot Q}\land R}}$
  3. $\paren{P\implies \paren{Q\implies P}}\iff Q$
練習問題2

次の式が命題論理における論理式であることを示せ．ただし，$P,Q$は原子命題とする．

  1. $P\implies P$
  2. $\lnot P\land Q$
  3. $P\iff P$
解答

定義通り順に確認していけばよい．

  1. 与式を変形する． \begin{align*} &P\implies P\\ \Iff&\implies\paren{P,P}\\ \end{align*} よって，$P$は原子命題より$P$は論理式である．$\implies$は2項真理関数で$P$は論理式より$\implies\paren{P,P}$は論理式である．したがって，$P\implies P$は論理式である．
  2. 与式を変形する． \begin{align*} &\lnot P\land Q\\ \Iff&\paren{\paren{\lnot P}\land Q}\\ \Iff&\land\paren{\lnot\paren{P},Q}\\ \end{align*} よって，$P$は原子命題より$P$は論理式である．$\lnot$は1項真理関数で$P$は論理式より$\lnot\paren{P}$は論理式である．また，$Q$は原子命題より$Q$は論理式である．よって，$\land$は2項真理関数で$\lnot\paren{P}$と$Q$は論理式より$\land\paren{\lnot\paren{P},Q}$は論理式である．したがって，$\lnot P\land Q$は論理式である．
  3. 与式を変形する． \begin{align*} &P\iff P\\ \Iff&\paren{P\implies P}\land\paren{P\implies P}\\ \Iff&\land\paren{P\implies P,P\implies P}\\ \end{align*} (1)より$P\implies P$は論理式である．$\land$は2項真理関数で$P\implies P$は論理式より$\land\paren{P\implies P,P\implies P}$は論理式である．したがって，$P\iff P$は論理式である．

量化子の略記

量化子は適用範囲を決める必要があり，その範囲を括弧で囲わなければならない．しかし，変項が増えると括弧が煩雑になってしまうため，省略を考える．他にも便利な記法として次のものを定める．

定義

量化子$\Lambda,\Lambda'$について，
適用範囲の略記
$$\Lambda x,\phi\Def\Lambda x\paren{\phi}$$
量化列挙の略記
$$\Lambda x\Lambda' y,\phi\Def\Lambda x,\Lambda' y,\phi$$
同量化の略記
$$\Lambda xy,\phi\Def\Lambda x\Lambda y,\phi$$
論理式$\phi$，$1$項述語$F\paren{x}$，項$x,y$について，
$\forall$の条件の略記
$$\forall x\paren{F\paren{x}},\phi\Def\forall x,F\paren{x}\implies\phi$$
$\exists$の条件の略記
$$\exists x\paren{F\paren{x}},\phi\Def\exists x\phi\land F\paren{x}$$
唯一存在量化子
$$\exists! x,F\paren{x}\Def\exists x\paren{F\paren{x}}\land\forall x\paren{F\paren{x}}y\paren{F\paren{y}},x=y$$

否定の略記

否定した命題や述語はよく使われる．そのたびに，先頭に$\lnot$をつけて括弧でくくっていては，冗長的で認識しづらい．量化子や関係演算子に斜線を入れることで否定を表現する．

定義

量化子$\Lambda$，変数$x,y$の関係演算子$\rho$について，
量化子の否定
$$\cancel{\Lambda} x,\phi\Def\lnot\paren{\Lambda x,\phi}$$
関係演算子の否定
$$x\,\cancel{\rho}\,y\Def\lnot\paren{x\,\rho\,y}$$

反転の略記

向きがある関係を表す演算子は表現力を上げるために逆向きで書くことがある．また，他の演算子の反転については適宜定義していく．

定義

論理式$\phi,\psi$について，
$\implies$の反転
$$\psi\rimplies\phi\Def \phi\implies \psi$$

推論

演繹と帰納

推論 inference とは論理的に説明することである．もうすこし具体的にいうと，前提 premise から結論 conclusion を論理的に導くことである．推論には大きく2つに分かれており，演繹と帰納である．演繹 deduction は前提を仮定して結論を得る方法である．このとき，結論のことを帰結 consequence という．演繹では使われた前提がすべて真であれば結論も真である．帰納 induction は前提からそれらしい一般的事実を導く方法である．結論が真とは限らないことに注意が必要である．また，言葉に帰納を含むが，対象をすべて網羅している場合である数学的帰納法などは演繹の一種である．しかし，数学の証明では演繹しか用いないので，演繹のことを単に推論といったり，数学的帰納法を単に帰納といったりもする．推論を次のように表す．

定義

任意の前提の集合$\Gamma$，帰結$\phi$としたとき，
$\Gamma$から$\phi$への推論
$$\Gamma\Implies\phi$$

論理的帰結

前提がすべて真であるとき，帰結が偽となりえない，または真となることが必然的であるとき，論理的帰結 logical consequence といい，次のように表す．

定義

命題の集合$\Gamma$，命題$\phi$としたとき，
$\Gamma$から$\phi$への論理的帰結
$$\Gamma\models\phi$$

証明

公理

推論では前提が真でなければ帰結が真であると言えない．このため，何かしらの命題を真であるとしないと何も始まらない，この命題を公理 axiom という．

推論規則

証明論では真偽の値を考えることをしない．そこで，どのような推論を許すかを定義することで真偽を定めずに帰結を判断できるようにした．この推論を推論規則 inference rule という．

証明体系

公理から推論規則を用いて新しい命題を推論していく．この公理と推論規則を定めた体系を証明体系 proof system という．公理と前提から推論規則で推論された命題を「前提から証明体系で演繹可能 deducible 」という．公理を空の前提から演繹可能であるとする．このとき，空の前提から推論規則で推論された命題を「証明体系で証明可能 proof system 」という．また，証明可能な命題を定理 theorem ，定理への演繹を証明 proof という．前提のうち，公理や定理でないものを仮定 assumption という．これを次のように表す．

定義

論理体系$\kappa$，命題の集合$\Gamma$，命題$\phi$としたとき，
$\kappa$で$\Gamma$から$\phi$は演繹可能
$$\Gamma\vdash_\kappa\phi$$
$\kappa$で$\phi$は証明可能
$$\vdash_\kappa\phi$$

証明体系が明らかなときは体系名を省略して$\vdash$を用いてよい．また，$a_1,a_2,\cdots,a_n$の集合を$\set{a_1,a_2,\cdots,a_n}$と表すが，集合の本来の文字を用いると，認識しづらくなるので次の表現を使う．

定義

命題の集合$\set{\phi_1,\phi_2,\cdots,\phi_n}$，命題$\phi$としたとき，
集合の省略
$$\phi_1,\phi_2,\cdots,\phi_n\Implies\phi\Def\set{\phi_1,\phi_2,\cdots,\phi_n}\Implies\phi$$ $$\phi_1,\phi_2,\cdots,\phi_n\models\phi\Def\set{\phi_1,\phi_2,\cdots,\phi_n}\models\phi$$ $$\phi_1,\phi_2,\cdots,\phi_n\vdash\phi\Def\set{\phi_1,\phi_2,\cdots,\phi_n}\vdash\phi$$
